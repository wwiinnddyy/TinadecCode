# 首页 Codex 风格侧边栏重构计划

## 问题分析

当前侧边栏（AppSidebar.vue）是"项目列表 + 会话列表"两个独立面板，不符合 Codex 应用的核心设计理念。

**Codex 应用的精髓**：以 **项目（Projects）** 和 **线程（Threads）** 为核心组织方式。侧边栏是一个树形结构，项目像文件夹一样可以展开，展开后显示该项目下的所有线程（对话）。顶部有"新建对话"和"搜索"两个快捷操作。

## Codex 侧边栏结构

```
┌──────────────────────────┐
│  [+ 新对话]   [🔍 搜索]  │  ← 快捷操作栏
├──────────────────────────┤
│  📁 my-app               │  ← 项目（可展开/折叠）
│    ├ 💬 修复登录 Bug      │  ← 线程/对话
│    ├ 💬 添加用户注册      │
│    └ 💬 重构 API 层       │
│  📁 tinadec-core          │
│    ├ 💬 更新依赖          │
│    └ 💬 迁移到 Elysia     │
│  📁 blog-site             │  ← 折叠状态
│                           │
│  ─────────────────────── │
│  [打开项目...]            │  ← 底部操作
└──────────────────────────┘
```

## 当前 vs 目标

| 维度 | 当前 | 目标（Codex 风格） |
|------|------|-------------------|
| 侧边栏结构 | 两个独立面板：项目 + 会话 | 树形结构：项目可展开，内含线程 |
| 新建对话 | 需先选项目，再点"新建会话" | 顶部快捷按钮，直接新建 |
| 搜索 | 无 | 顶部搜索按钮，搜索线程标题 |
| 项目展示 | 平铺列表 | 文件夹式，可展开/折叠 |
| 线程归属 | 独立面板，需手动关联 | 嵌套在所属项目下 |
| 顶部导航 | 标题栏放导航图标 | 侧边栏顶部放快捷操作 |

## 实施步骤

### 步骤 1：重构 AppSidebar.vue

将 AppSidebar 从"双面板"改为"树形侧边栏"：

1. **顶部快捷栏**：`[+ 新对话]` + `[🔍 搜索]` 两个按钮
2. **项目树**：每个项目是一个可展开/折叠的节点
   - 点击项目名 → 展开/折叠该项目下的线程列表
   - 展开后显示该项目所有线程（对话）
   - 点击线程 → 切换到该对话
3. **底部操作**：`[打开项目...]` 按钮

需要新增的状态：
- `expandedProjectIds: Set<string>` — 记录哪些项目是展开的
- `searchQuery: string` — 搜索关键词
- `showSearch: boolean` — 是否显示搜索框

### 步骤 2：更新 HomePage.vue

1. 移除对 `AppSidebar` 的 `select-project` 事件依赖
2. 侧边栏只需发出 `select-session` 和 `create-session` 事件
3. 点击项目名只展开/折叠，不切换项目
4. 点击线程才切换对话
5. "新建对话"按钮：如果有选中项目，在该项目下创建；否则弹出项目选择

### 步骤 3：更新 AppHeader.vue

1. 移除标题栏中的导航图标（聊天/项目），这些功能已移到侧边栏
2. 标题栏只保留：品牌标识 + 连接状态 + 语言切换 + 设置按钮
3. 更简洁，更接近 Codex 的标题栏风格

### 步骤 4：补充 i18n 翻译

新增翻译 key：
- `sidebar.newThread` — "新对话" / "New Thread"
- `sidebar.search` — "搜索" / "Search"
- `sidebar.searchPlaceholder` — "搜索对话..." / "Search threads..."
- `sidebar.noResults` — "无结果" / "No results"
- `sidebar.openProject` — 已有，保持

### 步骤 5：添加 CSS 样式

新增样式：
- `.sidebar-toolbar` — 顶部快捷操作栏（新对话 + 搜索）
- `.sidebar-search` — 搜索输入框
- `.project-tree-item` — 项目节点（文件夹样式）
- `.project-tree-item.expanded` — 展开状态
- `.thread-tree-item` — 线程节点（对话样式）
- `.thread-tree-item.active` — 选中状态
- `.sidebar-bottom` — 底部操作区

### 步骤 6：验证

- TypeScript 类型检查通过
- Vite 构建通过
- 侧边栏树形展开/折叠正常
- 搜索过滤正常
- 新建对话正常
- 切换对话正常
